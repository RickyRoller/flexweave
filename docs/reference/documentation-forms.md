# Documentation Forms

Every documentation page must choose one Diataxis form.

## Tutorial

A tutorial teaches through a complete learning path. It may use guided sample
content, but it should not become an API listing or rationale essay.

## How-To Guide

A how-to guide solves one operational task for a reader who already knows the
basic concepts.

## Reference

A reference page describes stable contracts, command names, file layouts, data
shapes, and package boundaries.

## Explanation

An explanation page records concepts, rationale, trade-offs, and product
boundaries.

## Rule

If a page starts mixing forms, split it into separate pages instead of adding
sections with different jobs.
