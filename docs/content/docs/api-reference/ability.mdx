Use checked grant and owner activation paths when an `ObjectStore` is available.
Lifecycle facts are emitted through executor sinks; stores do not publish them
to channels automatically.

## Lifecycle Shape

Activation emits attempted and then either started or rejected facts. Commit
emits committed facts after caller-owned commit behavior succeeds. End, cancel,
rollback, and revoke remove active state through separate commands so runtimes
can distinguish normal completion from cleanup or failure.

## Checked Paths

Use `AbilityGrant::checked` to reject invalid owners before a grant enters the
store. Use `AbilityActivation::for_owner` to reject owner mismatches before a
gate runs.

Registered definition paths validate definition keys without causing automatic
orchestration. Caller code still owns content loading and runtime behavior.
